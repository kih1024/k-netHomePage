각 기능 Layout

===관리자 페이지===
- 회원관리
└> 회원 삭제, 회원 권한 변경, 회원 비밀번호 등 개인정보 수정
└> 회원 권한 : 관리자, 비회원, [일반회원, 정회원], (세미나장, 세미나원)
  └> 각각 라디오, CheckButton 버튼으로 생성
일반회원 접근 권한 : 족보 제외
└> 관리자의 경우 관리자 계정을 만들어 하나를 임원진들이 공유하여 사용하는 식

- 게시판 관리
+ 게시판 추가의 경우 구현 하지 않을 예정
읽기 : 접근가능
쓰기 : 접근가능 + 글쓰기 버튼 생성 + 댓글가능
└> 읽기/쓰기 에 대해 일반회원을 정회원 하위개념으로 보고
일반, 관리자, 세미나원, 세미나장, 정회원 중 하나를 골라 선택하여 권한을 부여할 예정


===회원가입===
-회원 ID, 비밀번호, 비밀번호 확인, 실명, 성별, 생일, 기수, 전화번호, E-Mail, 학번(10자리 확인), 개인정보 동의
+ 기수, 생일(월)의 경우 Drop Down식으로 구현.
└> 생일(년, 일)의 경우 textbox 작성
+ 나머지 항목의 경우 직접 입력, 학번은 자릿수 확인
+ ID 중복확인 필요 (중복체크 필수)
└> ID(변경 불가), 실명(변경 불가) : 회색으로 background에 표시
+ 비밀번호의 경우 6자리 이상 (영문/숫자 혼합)
+ 개인정보 수집 약관 (가장 하단에 체크박스 식으로 배치)
└ NAVER의 회원가입 페이지 참조
+ 각 textbox의 경우 회색으로 background에 표시
+ 권한 : 관리자, 세미나 선생, 세미나 인원, 정회원

===개인정보 수정===
- 회원가입 Layout을 차용함
+ 비활성화 : ID, 실명
+ 활성화 : ID, 실명 제외 나머지 부분
+ 개인정보 수집 약관 없음

===ID/PW 찾기===
+ 회원가입시 사용된 E-Mail 입력 -> ID와 임시 PW를 해당 E-Mail로 전송
└> http://www.inmotionhosting.com/support/website/sending-email-from-site/using-the-php-mail-function-to-send-emails
참고 하여 e-mail 인증을 시도
+ DB에 해당 E-Mail이 없을 경우 에러 메시지 발송

===Fun&Issue 게시판===
- 상단에 해당 게시판 검색 란 View
http://imo.zz.am 참고
+ 글번호, 제목, 작성자, 작성일시, 조회수 순으로 글 View
└> 하단에 글쓰기

===익명 게시판===
-에브리타임 Layout 차용
+ 글 작성에 대해서는 바로 작성가능하도록 TextBox 작성
└> 사진 업로드 가능, 
+ 무조건적으로 익명으로 게시 (DB에는 작성자 저장)
+ 글이 길 경우 (더보기 생기도록) -> 각 글 칸을 맞추기 위해
+ 댓글은 가능하도록 구현
└> 자신이 쓴 글에 한해 수정, 삭제 가능, 글목록 버튼 만들어 이전으로 돌아갈 수 있도록 구현
+ 내용 검색 기능 추가

===글쓰기 Layout===
- 제목, 글쓰기 Editor
- 동영상 링크, 사진 삽입, 첨부파일
└> 동영상 링크 하단에 Editor에 링크 바로 삽입 가능한 안내 메시지 출력
+ 글삭제의 경우 로그인 세션과 해당 글의 작성자 같은지 확인
└> 해당 글 DB에 ID 보유하고 있어야함.


===K-Net 게시판===
- 로고, 주요활동, 사진, 연혁 소개 한페이지에 소개,
+ 임원진 사진 및 이름은 넣지않는 것으로
└> 공지사항에 임원진 소개를 넣는 것으로. (상단고정)

===기타 사항===
+ 답글의 경우 메인 답글에만 답변이 가능하도록 수정
└> 답글의 경우 항시 View
+ 자신의 답글에 한해 수정, 삭제 버튼 활성화
+ 게시글 창에 글삭제, 글수정, 목록, 글쓰기
